# 접수 PRD

## 1. 개요
- **모듈명**: 접수
- **목적**: 환자 내원 시 접수 처리, 신환 등록, 보험 확인, 대기 목록 관리
- **사용자**: 원무(주), 간호사

## 2. 사용자 시나리오

### 시나리오 1: 재진 환자 접수
```
1. 원무가 접수 화면에서 환자명 또는 차트번호로 검색한다
2. 환자를 선택하면 기본정보가 표시된다
3. 보험 확인 후 접수 버튼을 클릭한다
4. 환자가 대기 목록에 추가된다
5. 진료실에 대기 알림이 전송된다
```

### 시나리오 2: 신환 등록 + 접수
```
1. 원무가 '신환 등록' 버튼을 클릭한다
2. 환자 기본정보를 입력한다 (이름, 주민번호, 연락처, 보험정보)
3. 가족/보호자 정보를 입력한다
4. 동의서를 확인/서명 받는다
5. 등록 완료 후 바로 접수 처리한다
```

### 시나리오 3: 건강보험 자격 확인
```
1. 접수 시 주민번호로 건강보험 자격을 조회한다
2. 보험 유형(건강보험/의료급여/산재/자보)이 자동 표시된다
3. 자격이 없으면 경고를 표시한다
```

## 3. 화면 구성

### 3.1 접수 메인
- **카드 모듈**:
  - 환자 검색 (검색바 + 결과 테이블)
  - 접수 환자 정보 (기본정보 + 보험 + 바이탈 입력)
  - 대기 목록 (상태별 필터, 대기시간 표시)
  - 접수 이력 (오늘 접수 건 목록)

### 3.2 신환 등록 (모달/페이지)
- 기본정보 입력 폼
- 보험 정보
- 가족/보호자
- 동의서 체크
- 알러지/특이사항

## 4. 기능 명세

### 4.1 환자 검색
| 항목 | 설명 |
|------|------|
| 기능 설명 | 이름, 차트번호, 주민번호, 전화번호로 환자 검색 |
| 검증 규칙 | 최소 2글자 이상 입력 |
| 결과 | 일치 환자 목록 테이블 (이름, 차트번호, 생년월일, 최근 내원일) |

### 4.2 접수 처리
| 항목 | 설명 |
|------|------|
| 기능 설명 | 환자를 선택하여 접수 처리 |
| 입력 | 환자ID, 접수 유형(초진/재진), 진료과, 담당의 |
| 출력 | 접수 완료 + 대기 목록 추가 |
| 검증 | 이미 접수된 환자 중복 체크 |

## 5. 데이터 명세

### 5.1 접수 정보
| 필드명 | 타입 | 필수 | 설명 | 예시 |
|--------|------|------|------|------|
| patientId | string | Y | 환자 ID | P2026-00847 |
| visitType | enum | Y | 초진/재진 | 'first' / 'return' |
| department | string | Y | 진료과 | '내과' |
| doctor | string | Y | 담당의 | '오윤경' |
| insuranceType | enum | Y | 보험 유형 | 'national' / 'medical_aid' / 'industrial' / 'auto' |
| receptionTime | datetime | Y | 접수 시간 | 2026-04-05T14:30:00 |
| status | enum | Y | 상태 | 'waiting' / 'in_progress' / 'done' / 'cancelled' |

## 6. 연관 모듈
- 진료: 접수 완료 → 진료 대기 목록에 추가
- 수납: 접수 시 수가 기본 정보 생성
- 예약: 예약 환자 접수 시 예약 상태 변경
