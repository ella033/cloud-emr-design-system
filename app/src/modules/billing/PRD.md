# 수납 PRD

## 1. 개요
- **모듈명**: 수납
- **목적**: 진료 후 진료비 계산, 수납 처리, 영수증 출력, 미수금 관리
- **사용자**: 원무(주), 관리자

## 2. 사용자 시나리오

### 시나리오 1: 일반 수납
```
1. 진료 완료 후 환자가 수납 창구로 이동한다
2. 원무가 환자를 선택하면 진료비가 자동 계산되어 표시된다
3. 보험 급여/비급여 항목이 구분 표시된다
4. 결제 방법을 선택한다 (현금/카드/계좌이체)
5. 수납 처리 후 영수증을 출력한다
```

### 시나리오 2: 미수금 수납
```
1. 미수금 목록에서 환자를 검색한다
2. 미수금 내역과 금액을 확인한다
3. 부분 수납 또는 전체 수납을 처리한다
```

## 3. 화면 구성

### 3.1 수납 메인
- **카드 모듈**:
  - 수납 대기 목록 (진료 완료 환자)
  - 진료비 명세 (항목별 급여/비급여/본인부담)
  - 결제 처리
  - 수납 이력 (오늘 수납 건)
  - 미수금 목록

## 4. 데이터 명세

### 4.1 수납 정보
| 필드명 | 타입 | 필수 | 설명 | 예시 |
|--------|------|------|------|------|
| visitId | string | Y | 내원 ID | V2026-04-001 |
| totalAmount | number | Y | 총 진료비 | 26850 |
| insuranceCovered | number | Y | 보험 부담 | 18790 |
| patientCopay | number | Y | 본인 부담 | 8060 |
| paymentMethod | enum | Y | 결제 방법 | 'cash' / 'card' / 'transfer' |
| status | enum | Y | 상태 | 'pending' / 'paid' / 'partial' / 'unpaid' |

## 5. 연관 모듈
- 진료: 처방 데이터 → 수가 계산
- 접수: 보험 유형 참조
- 통계: 매출 데이터 전달
