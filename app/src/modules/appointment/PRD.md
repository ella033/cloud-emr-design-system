# 예약 PRD

## 1. 개요
- **모듈명**: 예약
- **목적**: 환자 예약 등록/조회/변경/취소, 캘린더 뷰 제공
- **사용자**: 원무, 간호사, 의사

## 2. 사용자 시나리오

### 시나리오 1: 예약 등록
```
1. 원무가 예약 화면에서 날짜/시간을 선택한다
2. 환자를 검색하여 선택한다
3. 진료 유형, 담당의, 메모를 입력한다
4. 예약 등록 후 환자에게 예약 확인 문자가 발송된다
```

### 시나리오 2: 예약 캘린더 확인
```
1. 의사가 주간/일간 캘린더에서 예약 현황을 확인한다
2. 빈 슬롯과 예약된 슬롯이 색상으로 구분된다
3. 예약을 클릭하면 환자 정보가 표시된다
```

## 3. 화면 구성

### 3.1 예약 메인
- **카드 모듈**:
  - 캘린더 뷰 (일간/주간/월간 전환)
  - 예약 목록 (테이블)
  - 예약 등록 폼 (사이드패널)
  - 의사별 일정 필터

## 4. 데이터 명세

### 4.1 예약 정보
| 필드명 | 타입 | 필수 | 설명 | 예시 |
|--------|------|------|------|------|
| patientId | string | Y | 환자 ID | P2026-00847 |
| date | date | Y | 예약 날짜 | 2026-04-10 |
| time | time | Y | 예약 시간 | 10:30 |
| doctor | string | Y | 담당의 | 오윤경 |
| visitType | enum | Y | 진료 유형 | 'first' / 'return' / 'checkup' |
| memo | string | N | 메모 | '혈압 재확인' |
| status | enum | Y | 상태 | 'scheduled' / 'confirmed' / 'cancelled' / 'no_show' |

## 5. 연관 모듈
- 접수: 예약 환자 도착 시 자동 접수
- 진료: 예약 정보 → 진료 화면 연동
- 홈: 오늘 예약 목록 표시
