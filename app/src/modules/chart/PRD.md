# 진료 PRD

## 1. 개요
- **모듈명**: 진료
- **목적**: 환자 진료의 모든 과정을 하나의 화면에서 수행. EMR의 핵심 모듈
- **사용자**: 의사(주), 간호사

## 2. 사용자 시나리오

### 시나리오 1: 일반 진료 흐름
```
1. 의사가 대기 환자 목록에서 환자를 선택한다
2. 환자 기본정보, 내원 이력, 알러지가 표시된다
3. 증상을 입력하고, 바이탈을 확인한다
4. 진단 코드(KCD)를 검색하여 입력한다
5. 처방을 입력한다 (약품 검색 → 용량/횟수/일수 설정)
6. 임상메모(SOAP)를 작성한다
7. 처방을 전송한다
8. 다음 환자로 넘어간다
```

### 시나리오 2: 묶음처방 사용
```
1. 퀵패널에서 '감기 기본 셋' 묶음처방을 클릭한다
2. 미리 설정된 처방이 자동으로 입력된다
3. 필요 시 개별 약품을 수정/삭제한다
```

### 시나리오 3: 알러지 경고
```
1. 처방 입력 시 DUR 체크가 자동 실행된다
2. 알러지 약물이 포함되면 경고 모달이 표시된다
3. 의사가 확인 후 계속 처방 또는 취소를 선택한다
```

## 3. 화면 구성

### 3.1 진료 메인
- **레이아웃**: Topbar + Sidebar(환자목록) + Main(카드 그리드) + QuickPanel
- **카드 모듈** (드래그/리사이즈 가능):
  - 환자 기본정보 (PRD 완성)
  - 내원 히스토리 (쓰레드형)
  - 바이탈 테이블 (PRD 완성)
  - 진단 및 처방
  - 임상메모 (SOAP)
  - 환자메모 (채팅)
  - 증상 입력
  - 검사 결과
  - 이미지 뷰어
  - 약물 타임라인

## 4. 기능 명세

### 4.1 진단 입력
| 항목 | 설명 |
|------|------|
| 기능 설명 | KCD 코드 또는 질환명으로 진단 검색/입력 |
| 입력 | 검색어 (코드 or 한글) |
| 출력 | 진단 목록 (주/부 구분) |
| 검증 | 최소 1개 주진단 필수 |

### 4.2 처방 입력
| 항목 | 설명 |
|------|------|
| 기능 설명 | 약품 검색 → 용량/횟수/일수/경로 설정 |
| 입력 | 약품명, 용량, 횟수, 일수, 투여경로 |
| 검증 | DUR 체크 (중복/금기/연령/임산부), 용량 범위 체크 |
| 에러 | 알러지 경고 모달, DUR 경고 모달 |

### 4.3 처방 전송
| 항목 | 설명 |
|------|------|
| 기능 설명 | 작성된 처방을 약국/검사실로 전송 |
| 검증 | 진단 1개 이상, 처방 1개 이상 필수 |
| 확인 | 전송 전 확인 모달 |

## 5. 데이터 명세

### 5.1 진단 정보
| 필드명 | 타입 | 필수 | 설명 | 예시 |
|--------|------|------|------|------|
| kcdCode | string | Y | KCD 코드 | J06.9 |
| diseaseName | string | Y | 질환명 | 급성 상기도감염 |
| isPrimary | boolean | Y | 주진단 여부 | true |

### 5.2 처방 정보
| 필드명 | 타입 | 필수 | 설명 | 예시 |
|--------|------|------|------|------|
| drugCode | string | Y | 약품 코드 | D001234 |
| drugName | string | Y | 약품명 | 타이레놀 500mg |
| dose | string | Y | 1회 용량 | 1T |
| frequency | number | Y | 1일 횟수 | 3 |
| days | number | Y | 투약 일수 | 3 |
| route | enum | Y | 투여 경로 | 'oral' / 'injection' / 'topical' |

## 6. 연관 모듈
- 접수: 대기 환자 데이터 수신
- 수납: 처방 전송 시 수가 데이터 전달
- 기초자료: 약품/질병코드 마스터 참조
